### 关键实现细节

```python
# should_explore 返回值规范
def should_explore(self, node: str, from_parent: Optional[str]) -> tuple[bool, str]:
    """
    Returns:
        (True, "first_visit")          → 新节点，需要探索
        (True, "not_yet_explored")     → 已存在但未探索，需要探索
        (False, "linked_only")         → 已探索，只更新关联
        (False, "already_explored")    → 已探索且已知此父节点，跳过
    """
```

- V2.4: `get_high_degree_unexplored` 返回度最高的未充分探索节点
- V2.5: `save()`/`load()` 正确持久化图结构

### 关键接口调用规范

```python
# Explorer.explore() - 正确调用方式
curiosity_item = {
    "topic": self.current_node,
    "score": 0.5,
    "depth": 2  # 对应 medium
}
findings = self.explorer.explore(curiosity_item)

# should_continue - 正确调用方式
should_continue, reason = self.controller.should_continue(self.current_node)
# 内部自动查 monitor.get_marginal_returns()

# decomposer.decompose - 正确调用方式
subtopics = await self.decomposer.decompose(self.current_node)
# 返回 list[dict]，每个 dict 包含 "topic" 字段
```

### 边际收益判断逻辑

```python
should_continue, reason = self.controller.should_continue(self.current_node)
if should_continue:
    # 深入：将子节点加入 frontier
    self._expand_frontier(self.current_node, findings)
    self.consecutive_low_gain = 0
else:
    # 跳转
    self.kg.mark_fully_explored(self.current_node)
    self.consecutive_low_gain += 1
    
    if self.consecutive_low_gain >= MAX_CONSECUTIVE_LOW:
        # 强制跳转
        forced = self.kg.get_high_degree_unexplored()
```

### 验收标准
- V1.1: `run_loop()` 启动后持续运行，不中断
- V1.2: `should_continue` 返回 False 时跳转
- V1.3: 连续 3 次低收益时强制跳转到 `get_high_degree_unexplored()`
- V1.4: frontier 为空时从 KG 选择节点
- V1.5: `explorer.explore()` 传 dict 参数
- V1.6: `decomposer.decompose()` 使用 await
