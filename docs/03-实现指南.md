## 四、核心代码解读

### 4.1 知识图谱状态生命周期

```
topic 进入知识图谱:
    add_knowledge() → known=True, depth=初始值
         ↓
    随着探索深入:
    add_knowledge() → depth=max(当前, 新值)
         ↓
    超过 max_knowledge_nodes:
    保留 depth 最高的，删除其他的
```

### 4.2 好奇心队列生命周期

```
添加: add_curiosity() → status=pending
         ↓
    被选中: select_next() → status=investigating
         ↓
    探索完成: explore() → status=done
         ↓
    重新激活: rescore_all() → 可能 status=pending 再次出现
```

### 4.4 评分算法详解

```python
def compute_curiosity_score(topic, base_relevance, base_depth):
    # 1. Relevance: 匹配用户兴趣
    relevance = base_relevance
    for interest in USER_INTERESTS:
        if any(kw in topic.lower() for kw in interest.split()):
            relevance += 1.5

    # 2. Recency: 多久没更新
    last_updated = knowledge_graph.get_last_updated(topic)
    hours_old = (now - last_updated).hours
    recency = min(10, hours_old / 24)

    # 3. Depth: 知识缺口
    depth = base_depth

    # 4. Surprise: 越新越惊喜
    known_count = count_related_nodes(topic)
    surprise = 10 if known_count == 0 else max(1, 10 - known_count * 2)

    # 加权求和
    return (relevance * 0.35 + recency * 0.25 +
            depth * 0.25 + surprise * 0.15)
```

import subprocess
import json

def curious_agent_run_one():
    """OpenCode 可调用此函数执行一轮探索"""
    result = subprocess.run(
        ["python3", "/root/dev/curious-agent/curious_agent.py", "--run"],
        capture_output=True, text=True, timeout=60
    )
    if result.returncode == 0:
        state = json.load(open("/root/dev/curious-agent/knowledge/state.json"))
        return {
            "topics_known": len(state["knowledge"]["topics"]),
            "curiosities_pending": sum(1 for i in state["curiosity_queue"] if i["status"] == "pending"),
            "last_exploration": state["exploration_log"][-1] if state["exploration_log"] else None
        }
    return None

## 七、常见问题

### Q: Web Search 返回空结果
A: 检查 `BOCHA_API_KEY` 是否设置，以及网络连通性。探索器会自动降级到推理模式。

### Q: 状态文件损坏
A: 备份并重建: `mv state.json state.json.bak; echo '{}' > state.json`

### Q: Cron 不执行
A: 检查 cron 服务状态: `systemctl status cron`，并查看日志: `grep CRON /var/log/syslog`

### Q: 探索结果质量低
A: 这是 Bocha API 中文优化的副作用。可以通过增加 `count` 参数或切换 API 改善。

### Q: 好奇心队列空了
A: `generate_initial_curiosities()` 会自动生成初始队列，或手动 `--inject` 添加。
