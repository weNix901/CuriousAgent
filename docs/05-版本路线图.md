# 版本路线图：Curious Agent

---

## v0.1 MVP ✅ (2026-03-19)

**目标**: 验证好奇驱动的自主探索可行性

### 已实现
- ✅ 好奇心队列 + 动态评分
- ✅ Bocha Web Search 探索
- ✅ 深度推理 fallback
- ✅ JSON 持久化知识图谱
- ✅ Cron 定时探索
- ✅ 状态面板 (`--status`)

### 技术栈
- Python 3.11
- Bocha Search API (`/v1/web-search`)
- JSON 文件持久化

---

## v0.2 增强版 (计划)

**目标**: 提高探索质量，增加用户可见性

### 功能
- [ ] **飞书通知集成** — 高价值发现自动推送
- [ ] **LLM 摘要生成** — 用 LLM 压缩长篇发现
- [ ] **探索策略多样化** — 深度探索 vs 广度探索
- [ ] **用户干预接口** — 暂停/优先/跳过好奇心项
- [ ] **SQLite 持久化** — 替换 JSON，提升查询性能

### 技术升级
- [ ] LlamaIndex 用于语义搜索
- [ ] 更可靠的搜索 API（Serper / Google）

---

## v0.3 多 Agent 版 (计划)

**目标**: 支持多 Agent 协作探索

### 功能
- [ ] **共享好奇心层** — 多个 Agent 共享探索队列
- [ ] **知识贡献追踪** — 记录每个 Agent 的贡献
- [ ] **冲突检测** — 不同 Agent 发现矛盾时告警
- [ ] **知识融合** — 多源发现的综合整理

### 架构
- [ ] Redis 用于跨进程状态共享
- [ ] 事件总线用于 Agent 间通信

---

## v1.0 完整版 (计划)

**目标**: 真正具备内在动机的自主 Agent

### 核心功能
- [ ] **内在动机引擎** — ICM (Intrinsic Curiosity Module) 实现
- [ ] **工作记忆建模** — Baddeley 多组件模型
- [ ] **自我改进循环** — 执行 → 评估 → 反思 → 策略更新
- [ ] **情感信号系统** — 好奇心/惊讶/满足的情绪化行为影响
- [ ] **长程规划** — 基于好奇心队列的年度/月度探索计划

### 集成
- [ ] OpenCode 深度集成
- [ ] OMO 多 Agent 系统联动
- [ ] 知识图谱可视化 (D3.js / Pyvis)

## 放弃列表 (Know What We're Not Doing)

- ❌ 图像/视频探索（超出 MVP 范围）
- ❌ 真正的强化学习训练（太重，先用启发式）
- ❌ 外部 API 付费依赖（坚持免费/已配置 API）
- ❌ 复杂的微服务架构（保持简单）
- ❌ 非结构化数据库（JSON → SQLite → 图数据库）
