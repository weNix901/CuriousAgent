# v0.2.9 — Agent 架构重构 + 知识结构化设计文档

> 设计时间：2026-04-12
> 版本目标：将 SpiderAgent/DreamAgent 重构为真正的 Agent（Nanobot 框架），同时建立结构化的知识表示

## 1. 设计概述

### 核心目标

1. **Agent 架构重构** — ExploreAgent 和 DreamAgent 成为真正的 Agent（ReAct 循环、Tool 接口）
2. **知识结构化** — 统一知识节点数据模型，支持热度、状态、版本化
3. **外部集成** — R1D3 可以通过 Skill 接口消费 CA 的知识

### 设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 实施策略 | 一次性完整实现 | 用户明确要求不分阶段 |
| KG 存储 | Neo4j + 迁移 | 从 JSON 迁移到 Neo4j |
| 测试策略 | TDD | RED-GREEN-REFACTOR，保证重构质量 |
| API 兼容 | 允许变化 | 追求最佳架构，不强制向后兼容 |
| LLM Provider | 多 Provider | 支持 volcengine、minimax 等切换 |

### 关键特性对比

| 特性 | ExploreAgent | DreamAgent |
|------|-------------|------------|
| **设计目的** | 探索未知知识入 KG | 检索（发现知识缺口） |
| **输入** | Queue 中的 topic | 全 KG 扫描 + 最近 7 天 ExplorationLogs |
| **输出** | 探索结果写入 KG | Queue topic（让 ExploreAgent 去加工） |
| **执行模型** | ReAct 循环（LLM 驱动） | 多周期循环（Light→Deep→Filtering→REM） |
| **Tool 数量** | 14 个 | 15 个 |
| **Daemon 模式** | 连续运行 | Heartbeat（6h 触发） |
| **KG 写入** | ✅ 写 KG | ❌ 不写 KG，只写 Queue |

## 5. DreamAgent 多周期架构

### 四层工序

| 层级 | 名称 | 职责 | 输入 | 输出 |
|------|------|------|------|------|
| L1 | Light Sleep | 候选筛选 | ExplorationLog + KG 异常节点 | 候选列表（≤100） |
| L2 | Deep Sleep | 信号评分 | 候选列表 | ScoredCandidate（top 20） |
| L3 | Filtering | 阈值门控 | ScoredCandidate | FilteredCandidate |
| L4 | REM Sleep | Queue 写入 | FilteredCandidate | Queue topic |

### 6 维评分信号

- Relevance (0.30) - 检索命中次数
- Frequency (0.24) - 探索日志出现次数
- Query Diversity (0.15) - 不同查询数
- Recency (0.15) - 时间衰减
- Consolidation (0.10) - 连续周期出现
- Conceptual Richness (0.06) - 概念标签密度

### 阈值门控

- minScore ≥ 0.80
- minRecallCount ≥ 3
- minUniqueQueries ≥ 3

## 6. Self-Evolution 引擎

### 核心流程

```
ExploreAgent 探索完成 → ExplorationLog → 贝叶斯更新权重 → 下次探索使用新策略
```

### EvolutionLog Schema

- topic, topic_type, tools_used
- duration_s, quality_score
- success, error_reason
- strategy_version, created_at
