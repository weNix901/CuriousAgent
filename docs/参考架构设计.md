# Curious Agent 参考架构设计

> **版本**: v0.2.6 | **最后更新**: 2026-03-29 (三代理并发架构 + 自由梦境) | **文档状态**: 参考架构

---

## 一、架构设计目标

Curious Agent 是一个具有**内在好奇心**、**元认知能力**和**自我进化**能力的自主 Agent 系统。架构设计的核心目标：

1. **主动探索**：不等待用户提问，自主发现知识缺口
2. **元认知调控**：知道自己懂什么、不懂什么，智能评估探索价值
3. **自我进化**：将发现转化为行为规则，持续改进自身
4. **可观测性**：所有状态可查、可调试、可追踪

#### 3.2.6.5 NodeLockRegistry

**文件**: `core/node_lock_registry.py`

**职责**: 提供线程安全的节点级锁定机制

**接口规约**:

```python
class NodeLockRegistry:
    """
    节点锁注册表。
    
    两层锁定机制:
    1. 全局写锁: 所有写操作必须获取，保证写操作互斥
    2. 节点级锁: 单个节点的细粒度锁，允许并发访问不同节点
    
    使用 WeakValueDictionary 自动清理未使用的锁。
    """
    
    _global_write_lock = threading.Lock()
    _node_locks = WeakValueDictionary()
    
    @classmethod
    def global_write_lock(cls) -> threading.Lock:
        """获取全局写锁"""
    
    @classmethod
    def get_lock(cls, node_id: str) -> threading.Lock:
        """
        获取指定节点的锁。
        
        Args:
            node_id: 节点标识符
            
        Returns:
            该节点的专属锁（每个节点一个锁实例）
        """
```

**使用模式**:

```python

    data = kg.get_topic_data(topic)

# 写入操作（需要全局锁 + 节点锁）

    lock = NodeLockRegistry.get_lock(topic)
```

```python
def get_spreading_activation_trace(
    topic: str,
    max_depth: int = 10,
    decay: float = 0.5,
    threshold: float = 0.1
) -> dict:
    """
    使用扩散激活算法从任意 topic 追溯根技术。

    机制：
    - 从起点 topic 开始，激活值 = 1.0
    - 激活沿所有边（parents + explains）并行扩散
    - 每跳衰减系数 = decay（默认 0.5）
    - 多条路径汇聚时，激活值累加（跨子图连接自然体现）
    - 激活值低于 threshold 时停止扩散
    - 按激活值降序排列，paths >= 2 的节点为候选根技术

    Returns:
        {
            "origin": str,
            "activation_map": dict,
            "ordered_trace": list[dict],  # 按激活值降序
            "root_technologies": list[dict],
            "cross_subgraph_connections": list[dict]
        }
    """

                            confidence: float = 0.8) -> None:
    """双向写入父子关系（parents + explains）"""
```

#### 扩散激活算法详解

```
算法：Collins & Loftus (1975) 扩散激活传播

初始化：
  activation[origin] = 1.0
  所有其他节点 = 0.0

扩散步骤（并行）：
  for each activated node N:
      for each neighbor M (via parents + explains):
          if M not visited:
              activation[M] += activation[N] * decay
              push M to queue

终止条件：
  - 激活值 < threshold（默认 0.1）
  - 或达到 max_depth（默认 10）

根技术判断：
  is_root = (topic in root_pool) OR (paths >= 2)

根评分公式：
  root_score = cross_domain_count * 0.4 + explains_count * 0.6
```

```python
import time
import hashlib
import requests
from threading import Thread

class CuriousHeartbeatClient:
    """
    Curious Agent心跳客户端
    
    功能：
    1. 定期健康检查（30秒间隔）
    2. 检测行为规则文件变更
    3. 增量同步新发现
    """
    
    def __init__(self, api_endpoint: str = "http://localhost:4848"):
        self.api_endpoint = api_endpoint
        self.last_behaviors_hash = None
        self.last_exploration_time = None
        self.running = False
        self.handlers = {
            "behaviors_updated": [],
            "new_discovery": [],
            "status_change": []
        }
    
    def start(self):
        """启动心跳线程"""
        self.running = True
        self.heartbeat_thread = Thread(target=self._heartbeat_loop, daemon=True)
        self.heartbeat_thread.start()
        print("[CuriousHeartbeat] Started")
    
    def stop(self):
        """停止心跳"""
        self.running = False
        print("[CuriousHeartbeat] Stopped")
    
    def _heartbeat_loop(self):
        """心跳主循环 - 每30秒执行一次"""
        while self.running:
            try:
                self._check_health()
            except Exception as e:
                print(f"[CuriousHeartbeat] Error: {e}")
                self._emit("status_change", {"status": "disconnected"})
            
            time.sleep(30)  # 30秒间隔
    
    def _check_health(self):
        """执行健康检查"""
        response = requests.get(
            f"{self.api_endpoint}/api/curious/health",
            timeout=5
        )
        
        if response.status_code != 200:
            return
        
        data = response.json()
        heartbeat = data.get("heartbeat_data", {})
        
        # 1. 检查行为规则更新
        current_hash = heartbeat.get("behaviors_version")
        if current_hash and current_hash != self.last_behaviors_hash:
            if self.last_behaviors_hash is not None:
                # 不是首次，触发增量加载
                self._emit("behaviors_updated", {
                    "old_version": self.last_behaviors_hash,
                    "new_version": current_hash
                })
            self.last_behaviors_hash = current_hash
        
        # 2. 检查新探索完成
        last_exploration = heartbeat.get("last_exploration")
        if last_exploration and last_exploration != self.last_exploration_time:
            if self.last_exploration_time is not None:
                # 有新探索完成，获取详情
                self._fetch_new_discoveries()
            self.last_exploration_time = last_exploration
        
        # 3. 状态变更通知
        self._emit("status_change", {
            "status": data.get("status"),
            "pending_queue": heartbeat.get("pending_queue_size", 0)
        })
    
    def _fetch_new_discoveries(self):
        """获取新完成的探索"""
        response = requests.get(
            f"{self.api_endpoint}/api/curious/state",
            timeout=5
        )
        
        if response.status_code == 200:
            state = response.json()
            recent = state.get("recent_explorations", [])
            
            for exploration in recent:
                if exploration.get("behavior_applied"):
                    self._emit("new_discovery", exploration)
    
    def on(self, event_type: str, handler: callable):
        """注册事件处理器"""
        if event_type in self.handlers:
            self.handlers[event_type].append(handler)
    
    def _emit(self, event_type: str, payload: dict):
        """触发事件"""
        for handler in self.handlers.get(event_type, []):
            try:
                handler(payload)
            except Exception as e:
                print(f"[CuriousHeartbeat] Handler error: {e}")


# 使用示例
class OpenClawAgent:
    def __init__(self):
        self.curious_heartbeat = CuriousHeartbeatClient()
        
        # 注册事件处理器
        self.curious_heartbeat.on("behaviors_updated", self._on_behaviors_update)
        self.curious_heartbeat.on("new_discovery", self._on_new_discovery)
        self.curious_heartbeat.on("status_change", self._on_status_change)
    
    def start(self):
        """启动Agent"""
        # 1. 首次加载所有行为规则
        self.load_curious_behaviors()
        
        # 2. 启动心跳监控
        self.curious_heartbeat.start()
    
    def _on_behaviors_update(self, payload: dict):
        """行为规则更新回调"""
        print(f"[Curious] Behaviors updated: {payload['new_version'][:8]}")
        # 增量加载新规则
        self.reload_curious_behaviors()
        # 可选：通知用户

    def _on_new_discovery(self, payload: dict):
```

## 十、设计原则总结

### 10.1 系统设计原则

1. **单一职责**: 每个模块只做一件事，接口清晰
2. **可观测性**: 所有状态持久化，可查询、可追踪
3. **可扩展性**: 通过注册中心、事件总线支持插件化扩展
4. **容错性**: 多级Fallback，优雅降级
5. **元认知**: 系统自我监控、自我调节
6. **进化性**: 发现→规则→行为→能力的正向循环

### 10.2 外部Agent集成原则

7. **能力解耦**: Curious Agent专注于"学习"，外部Agent专注于"执行"
8. **文件系统优先**: 行为规则通过文件共享，确保无依赖启动
9. **异步松耦合**: 事件通知机制，不阻塞外部Agent主流程
10. **语义可发现**: 记忆片段支持语义检索，而非精确匹配
11. **渐进增强**: 外部Agent可选择性集成，从只读到全双工

### 10.3 集成最佳实践

```
✅ 推荐做法:
   • 外部Agent启动时加载 behaviors.md
   • 定期(5分钟)检查文件更新
   • 通过语义相似度检索记忆
   • 异步注入话题，不等待响应
   • 订阅事件获取实时通知

❌ 避免做法:
   • 每次对话都重新加载全部规则
   • 阻塞等待Curious Agent探索完成
   • 直接读取和操作 state.json
   • 频繁轮询API (每秒多次)
   • 忽略API错误，不降级处理
```

## 十二、v0.2.6 Bug 修复与分解闭环架构

### 12.1 问题背景

v0.2.6 之前，Curious Agent 存在根本性的架构缺陷：

1. **分解能力缺失**: SpiderAgent 只探索，不分解，KG 永远无法形成树状结构
2. **闭环未形成**: DreamAgent 生成的 insights 只存在文件里，没有触发新探索
3. **写入逻辑分散**: decomposition 和 KG 写入分散在多处，导致不一致

### 12.2 架构修复方案

#### 12.2.1 统一分解写入 (Fix #6, #7)

**问题**: 分解逻辑在 `run_one_cycle()` 和 `SpiderAgent` 中重复，KG 写入不一致

**解决方案**: 新增 `decompose_and_write()` 统一入口

```
调用方:
  ├── curious_agent.run_one_cycle()
  ├── SpiderAgent._explore_topic()
  ├── SpiderAgent._process_inbox_cycle()
  └── curious_api.api_run()
           │
           ▼
    decompose_and_write(topic)
           │
           ├── decompose(topic) → subtopics
           │
           ├── kg.update_curiosity_status(topic, "exploring") — Fix #7
           │
           └── for each subtopic:
                   ├── kg.add_child(topic, subtopic)
                   └── kg.add_curiosity(subtopic, original_topic=topic)
```

**关键修复**:
- Fix #6: 分解和写入集中到单一方法
- Fix #7: 分解后立即标记 parent 为 exploring，确保 `mark_topic_done()` 能找到 parent

#### 12.2.2 SpiderAgent 集成 Decomposition (Fix #1, #2)

**问题**: SpiderAgent 只调用 `explorer.explore()`，没有任何 decomposition 逻辑

**修复后工作流**:

```
SpiderAgent._process_inbox_cycle()
    │
    ├── fetch_and_clear_dream_inbox() → inbox_items
    │
    └── for each item:
            ├── _explore_topic(topic) → result
            │       └── explorer.explore(curiosity_item)
            │
            ├── _decompose_and_enqueue(topic) — Fix #1 新增
            │       └── decomposer.decompose_and_write(topic)
            │               → subtopics
            │               → kg.add_child() / kg.add_curiosity()
            │
            └── _notify_dream_agent(topic, result)
                    → notification_queue.put()
                        → DreamAgent 下一轮处理
```

**闭环形成** (Fix #2):
```
DreamAgent 生成 insight
    ↓
kg.add_to_dream_inbox(trigger_topic)
    ↓
SpiderAgent 下一轮消费 inbox
    ↓
探索 trigger_topic + decomposition
    ↓
新 children 写入 KG
    ↓
DreamAgent 看到新 children，生成新 insight
    ↓
... 循环继续
```

#### 12.2.3 父子关系追踪修复 (Fix #7)

**问题**: `mark_topic_done()` 查找 parent 的逻辑错误

**原逻辑（错误）**:
```python
# 查找 status == "exploring" 的 queue item
for queue_item in state["curiosity_queue"]:
    if queue_item["status"] == "exploring":
        parent_topic = queue_item["topic"]  # Bug: topic 不是 parent 字段！
```

**修复后逻辑**:
```python
# 从当前 topic 的队列条目读取 original_topic
for item in state["curiosity_queue"]:
    if item["topic"] == topic:
        parent_topic = item.get("original_topic")  # Fix #6 存储的 parent
        break

if parent_topic:
    _update_parent_relation(parent_topic, topic)
```

**数据流**:
```
decompose_and_write(topic) 分解出 subtopic
    ↓
kg.add_curiosity(subtopic, original_topic=topic) — 存储 parent 关系
    ↓
SpiderAgent 探索 subtopic
    ↓
mark_topic_done(subtopic) 读取 original_topic → 找到 parent
    ↓
_update_parent_relation(parent, subtopic) 双向写入
```

#### 12.2.4 引用提取架构 (Fix #3, #4)

**论文引文提取** (Fix #3):

```
Explorer._layer2_arxiv(topic, arxiv_links)
    │
    ├── analyzer.analyze_papers(topic, arxiv_links) → papers
    │
    └── if papers:
            └── PaperCitationExtractor.extract_all(topic, papers)
                    │
                    ├── LLM 分析论文 abstract → 提取引文
                    ├── (可选) PDF 解析 References
                    └── 去重后返回 citations
                    │
                    └── for each citation:
                            ├── kg.add_citation(topic, citation_name)
                            └── kg.add_curiosity(citation_name, topic_type="citation")
```

**网页引用提取** (Fix #4):

```
Explorer._explore_layers(topic)
    │
    ├── _layer1_search(topic) → search_results, sources
    │
    └── WebCitationExtractor.extract_from_sources(topic, sources)
            │
            ├── for each source URL (max 3):
            │       └── _fetch_page(url) → content
            │       └── LLM 提取引用的技术/概念
            │
            └── for each citation:
                    ├── kg.add_child(topic, citation_name)
                    └── kg.add_curiosity(citation_name, topic_type="web_citation")
```

#### 12.2.5 去重逻辑修复 (Fix #8)

**问题**: 已完成的 topic 会被重复添加到队列

**原逻辑（错误）**:
```python
# 只跳过未完成的
def add_curiosity(topic, ...):
    for item in state["curiosity_queue"]:
        if item["topic"] == topic and item["status"] != "done":  # Bug!
            return
```

**修复后**:
```python
# 只要存在就跳过
def add_curiosity(topic, ...):
    for item in state["curiosity_queue"]:
        if item["topic"] == topic:  # 无论什么 status 都跳过
            return
```
