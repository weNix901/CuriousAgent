# Curious Agent v0.2.4 可持续架构实施计划

> **文档状态**: 已评审，基于长期架构视角重新设计  
> **版本**: v0.2.4  
> **日期**: 2026-03-25  
> **架构原则**: 单一事实源、向后兼容、渐进增强、接口优先

---

## 一、架构问题诊断

### 1.1 识别的架构反模式

| 反模式 | 风险等级 | 影响 | 原设计中的体现 |
|--------|----------|------|----------------|
| 数据模型分裂 | 🔴 高 | 维护成本倍增 | 新建 kg_graph.py 与现有 knowledge_graph.py 并存 |
| 状态分散 | 🔴 高 | 数据不一致 | Checkpoint、KG、state.json 分散存储 |
| 文件轮询通信 | 🔴 高 | 延迟高、难扩展 | R1D3Watcher 轮询文件系统 |
| 隐式依赖链 | 🟠 中 | 调试困难 | SpiderEngine 直接创建 Explorer/Decomposer |
| 硬编码决策 | 🟠 中 | 无法动态调整 | MIN_MARGINAL_RETURN=0.3 等常量 |
| 紧耦合存储 | 🟠 中 | 无法切换存储 | KGGraph 直接操作 JSON 文件 |
| 缺乏 Schema 版本 | 🟠 中 | 升级困难 | 无版本字段 |

### 1.2 长期风险预测

如果按照原设计直接实施：
- **3 个月后**：KG 双轨制导致数据不一致，需要花 1 周修复
- **6 个月后**：需要支持 Redis 存储，但存储格式与业务逻辑绑定，重构成本 2 周
- **12 个月后**：需要支持动态探索策略，但逻辑硬编码，需要大规模重构

**本计划的目标**：现在多花 3 天时间建立正确架构，避免未来 3 周的技术债务偿还。

### 2.1 决策记录 (Architecture Decision Records)

#### ADR-1: 统一知识图谱而非双轨制

**状态**: 已接受  
**背景**: 原设计新建 kg_graph.py，与现有 knowledge_graph.py 并存

**决策**: 扩展现有 knowledge_graph.py 支持图结构

**理由**:
- 单一事实源：所有 topic 关系在一个地方维护
- 向后兼容：现有代码无需修改
- 降低认知负担：开发者只需了解一套 API

**实现方式**:
```python
# 新增方法（图结构）
def add_relation(from_topic, to_topic, relation_type)
def get_topic_graph(topic) -> {parents, children, relations}

# 保留现有方法（向后兼容）
def add_child(parent, child)  # 内部调用 add_relation
```

**后果**:
- ✅ 避免数据不一致
- ✅ 降低维护成本
- ⚠️ 需要修改现有 knowledge_graph.py（风险可控）

#### ADR-4: 事件驱动 + 轮询 Fallback

**状态**: 已接受  
**背景**: 原设计纯文件轮询

**决策**: 优先使用事件总线，但保留文件轮询作为 fallback

**架构**:
```
Primary: EventBus (内存/Redis)
    R1D3 ──publish──► EventBus ──subscribe──► Spider
    
Fallback: File System
    R1D3 ──write file──► disk ──poll──► Spider
```

**理由**:
- 事件驱动延迟低、可扩展
- 文件系统简单可靠，作为 fallback 保证可用性
- 两者可并存，渐进迁移

#### 任务 4.3: SpiderEngine 重构 (2天)

**交付物**: `spider_engine.py` (重写)

**设计**:
```python
class SpiderEngine:
    def __init__(
        self,
        repo: KnowledgeRepository,
        explorer: TopicExplorer,
        decomposer: TopicDecomposer,
        controller: MetaCognitiveController,
        strategy: FrontierStrategy,
        config: SpiderConfig,
        checkpoint: SpiderCheckpoint | None = None,
    ):
        self.repo = repo
        self.explorer = explorer
        self.decomposer = decomposer
        self.controller = controller
        self.strategy = strategy
        self.config = config
        self.checkpoint = checkpoint
        
        # 运行时状态
        self.runtime_state = SpiderRuntimeState()
        
        # 从 checkpoint 恢复
        if checkpoint:
            self._restore_from_checkpoint()
    
    async def run_once(self) -> None:
        """单次探索循环"""
        # 1. 选择节点
        if not self.runtime_state.current_node:
            self._select_next_node()
        
        if not self.runtime_state.current_node:
            return  # 无可用节点
        
        node = self.runtime_state.current_node
        
        # 2. 探索
        result = await self.explorer.explore(node, self.config.default_exploration_depth)
        
        # 3. 评估质量
        quality = self.controller.monitor.assess_exploration_quality(node, {
            "summary": result.findings,
            "sources": result.sources,
            "papers": result.papers,
        })
        
        # 4. 边际收益判断
        should_continue, reason = self.controller.should_continue(node)
        
        # 5. 更新 KG
        topic = self.repo.get_topic(node) or Topic(name=node)
        topic.explored = True
        topic.last_quality = quality
        topic.findings_summary = result.findings[:500]
        self.repo.save_topic(topic)
        
        # 6. 扩展前沿
        if should_continue:
            await self._expand_frontier(node, result)
            self.runtime_state.consecutive_low_gain = 0
        else:
            topic.fully_explored = True
            self.repo.save_topic(topic)
            self.runtime_state.consecutive_low_gain += 1
            
            if self.runtime_state.consecutive_low_gain >= self.config.max_consecutive_low_gain:
                # 强制跳转
                next_node = self.repo.get_high_degree_unexplored()
                self.runtime_state.current_node = next_node
                self.runtime_state.consecutive_low_gain = 0
        
        # 7. 记录访问
        self.runtime_state.visited.add(node)
        self.runtime_state.step_count += 1
        
        # 8. Checkpoint
        if self.checkpoint:
            self.checkpoint.save(self.runtime_state, self.repo.get_storage_path())
    
    def _select_next_node(self) -> None:
        """选择下一个探索节点"""
        if self.runtime_state.frontier:
            self.runtime_state.current_node = self.runtime_state.frontier.pop(0)
        else:
            # 使用策略选择
            self.runtime_state.current_node = self.strategy.select_next(
                self.runtime_state.frontier,
                self.repo
            )
    
    async def _expand_frontier(self, node: str, result: ExplorationResult) -> None:
        """从探索结果扩展前沿"""
        subtopics = await self.decomposer.decompose(node)
        
        for st in subtopics[:5]:  # 最多 5 个
            child_name = st.get("topic", "")
            if not child_name:
                continue
            
            # 检查是否需要探索
            child = self.repo.get_topic(child_name)
            if not child:
                # 新节点
                child = Topic(name=child_name)
                self.repo.save_topic(child)
                self.repo.add_relation(node, child_name, "decomposed_from")
                self.runtime_state.frontier.append(child_name)
            elif not child.explored:
                # 已存在但未探索
                self.runtime_state.frontier.append(child_name)
            else:
                # 已探索，只建立关系
                self.repo.add_relation(node, child_name, "related_to")
```

```python
            # 事件驱动模式
            event_bus.subscribe("proposition_created", self._on_proposition)
    
    def _on_proposition(self, event: Event) -> None:
        """事件回调"""
        self._propositions.append(event.data)
    
    def poll_propositions(self) -> list[dict]:
        """
        轮询 fallback
        返回新命题列表
        """
        # 原有的文件轮询逻辑
        pass
    
    def get_new_propositions(self) -> list[dict]:
        """
        获取新命题
        优先返回事件接收的，如果事件不可用则轮询
        """
        if self.event_bus and self.event_bus.is_connected():
            # 返回事件队列中的
            propositions = self._propositions.copy()
            self._propositions.clear()
            return propositions
        else:
            # Fallback 到轮询
            return self.poll_propositions()
```

### 4.4 断路器模式 (Circuit Breaker)

```python
class ResilientLLMClient:
    """带断路器的 LLM 客户端"""
    
    def __init__(self, llm: LLMClient):
        self.llm = llm
        self.failure_count = 0
        self.threshold = 5
        self.is_open = False
    
    def chat(self, prompt: str) -> str:
        if self.is_open:
            raise CircuitBreakerOpen("LLM service unavailable")
        
        try:
            result = self.llm.chat(prompt)
            self.failure_count = 0
            return result
        except Exception as e:
            self.failure_count += 1
            if self.failure_count >= self.threshold:
                self.is_open = True
            raise
```
