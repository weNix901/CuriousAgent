### 2.2 Cognitive Answer Loop (Expected Behavior)

This is the **target behavior** after OpenClaw integration (v0.3.1+):

```
User Question
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│ OpenClaw: curious_cognitive_hook.before_turn()              │
│                                                             │
│ POST /api/knowledge/check                                   │
│ → returns: confidence, level, gaps, guidance                │
└─────────────────────────────────────────────────────────────┘
      │
      ▼
┌─────────────────────────────────────────────────────────────┐
│ confidence ≥ 0.6 → R1D3 answers from KG (cited sources)     │
│ → DONE                                                      │
└─────────────────────────────────────────────────────────────┘
      │ confidence < 0.6
      ▼
┌─────────────────────────────────────────────────────────────┐
│ R1D3 performs web search (Step 2)                           │
│                                                             │
│ If found → POST /api/knowledge/record                       │
│ → R1D3 answers from search + says "learning..."             │
│ → DONE                                                      │
└─────────────────────────────────────────────────────────────┘
      │ Not found / insufficient
      ▼
┌─────────────────────────────────────────────────────────────┐
│ R1D3 answers from LLM knowledge                             │
│                                                             │
│ → OpenClaw: curious_cognitive_hook.after_turn()             │
│ → POST /api/knowledge/learn                                 │
│ → CA injects topic to Queue for exploration                 │
│ → Next time: topic will be in KG                            │
└─────────────────────────────────────────────────────────────┘
```

## 3. API Design

### 3.1 Endpoint Specification

| Endpoint | Method | Purpose | Request | Response |
|----------|--------|---------|---------|----------|
| `/api/knowledge/check` | POST | Query KG confidence | `{topic: string}` | `{confidence, level, gaps, guidance}` |
| `/api/knowledge/learn` | POST | Inject unknown topic | `{topic, context, strategy, priority}` | `{success, queue_id}` |
| `/api/knowledge/analytics` | GET | Get interaction stats | — | `{kg_hits, search_hits, llm_fallbacks, topics_learned}` |
| `/api/knowledge/record` | POST | Record search result | `{topic, content, sources}` | `{success, node_id}` |

### 3.2 Response Schema Details

#### `/api/knowledge/check` Response

```json
{
  "success": true,
  "result": {
    "topic": "FlashAttention",
    "confidence": 0.45,
    "level": "novice",  // novice | beginner | intermediate | expert
    "gaps": ["implementation details", "benchmark results"],
    "guidance": "🟠 KG has limited knowledge. Search the web first. If found → record to KG + inject to CA. If not found → answer from LLM knowledge + inject to CA (mark as uncertain)."
  }
}
```

#### `/api/knowledge/learn` Response

```json
{
  "success": true,
  "result": {
    "queue_id": "q_abc123",
    "topic": "FlashAttention",
    "status": "pending",
    "estimated_exploration": "30-60 minutes"
  }
}
```

### 4.2 Guidance Message Templates

| Confidence | Level | Message Template |
|------------|-------|-----------------|
| ≥ 0.85 | Expert | 🟢 Answer from KG knowledge. Cite sources. No need to search or guess. |
| 0.6 - 0.85 | Intermediate | 🟡 Partial KG knowledge. Supplement with web search. Record findings. Inject to CA for deep exploration. |
| 0.3 - 0.6 | Beginner | 🟠 KG has limited knowledge. Search the web first. If found → record to KG + inject to CA. If not found → answer from LLM + inject to CA (mark as uncertain). |
| < 0.3 | Novice | 🔴 No KG knowledge. Search the web. Then answer from LLM if search fails. ALWAYS inject this topic to CA for exploration. Next time it will be in KG! |

## 8. Risks & Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| KG query latency slows API response | Medium | Cache confidence results, async stats recording |
| Topic extraction from questions inaccurate | Low | OpenClaw wrapper handles this (not CA scope) |
| Queue explodes with too many injections | High | Deduplication + rate limiting in inject_unknown() |
| Stats store grows unbounded | Medium | TTL-based cleanup, aggregation for old records |
