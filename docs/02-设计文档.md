# 设计文档：Curious Agent 技术架构

> **版本**: v0.1 | **状态**: 已实现

---

## 一、设计原则

1. **最小可行** — MVP 只做一件事：持续探索并积累知识
2. **状态显式化** — 所有状态（好奇心队列、知识图谱）必须可查、可调试
3. **评分可解释** — 每条好奇心的分数必须能追溯到具体来源
4. **优雅降级** — 无网络时能用推理模式继续工作

## 三、核心组件详解

### 3.1 知识图谱 (knowledge_graph.py)

**职责**：管理 Agent 的知识状态——什么已知、什么未知。

**状态模型**：

```json
{
  "version": "1.0",
  "last_update": "ISO timestamp",
  "knowledge": {
    "topics": {
      "topic_name": {
        "known": true,
        "depth": 0-10,
        "last_updated": "ISO",
        "sources": ["url1", "url2"],
        "summary": "..."
      }
    }
  },
  "curiosity_queue": [
    {
      "topic": "metacognition in AI",
      "score": 8.5,
      "reason": "深度理解用户研究方向",
      "relevance": 9.0,
      "depth": 8.0,
      "created_at": "ISO",
      "status": "pending|investigating|done|paused"
    }
  ],
  "exploration_log": [
    {
      "timestamp": "ISO",
      "topic": "...",
      "action": "web_search|inference",
      "findings": "...",
      "notified_user": true
    }
  ]
}
```

**设计决策**：
- 用 `status` 字段控制生命周期：`pending → investigating → done`
- `depth` 表示理解深度（0-10），探索后累加
- 限制节点数（默认100），保留深度最高的

### 3.2 好奇心引擎 (curiosity_engine.py)

**职责**：决定"什么值得探索"。

**评分公式**：

```
Score = Relevance × 0.35 + Recency × 0.25 + Depth × 0.25 + Surprise × 0.15
```

**各维度说明**：

| 维度 | 计算方式 | 范围 | 权重 |
|------|---------|------|------|
| **Relevance** | 匹配用户兴趣关键词 | 0-10 | 35% |
| **Recency** | 知识图谱中多久未更新 | 0-10 | 25% |
| **Depth** | 知识缺口深度 | 0-10 | 25% |
| **Surprise** | 知识图谱中已有多少相关节点 | 1-10 | 15% |

**Relevance 匹配示例**：
```python
USER_INTERESTS = [
    "AI agent autonomy",
    "metacognition",
    "curiosity-driven exploration",
    ...
]
# 如果 topic 包含关键词，Relevance +1.5
```

**Recency 计算**：
```python
# 每24小时 +1 分，上限10
hours_old = (now - last_updated).total_seconds() / 3600
recency = min(10.0, hours_old / 24)
```

**Surprise 计算**：
```python
# 知识图谱中越少相关节点，越惊喜
known = count_related_nodes(topic)
surprise = 10.0 if known == 0 else max(1.0, 10.0 - known * 2)
```

### 3.3 探索器 (explorer.py)

**职责**：执行实际的知识获取。

**工作流**：

```
选择好奇心项
    ↓
尝试 Bocha Web Search
    ↓ (成功)
综合发现 → 更新知识图谱 → 判断是否通知
    ↓ (失败)
深度推理 (基于已有知识推导) → 更新知识图谱
```

**Web Search 结果解析**：

Bocha API 返回格式：
```json
{
  "code": 200,
  "data": {
    "webPages": {
      "value": [
        {
          "name": "Title",
          "snippet": "...",
          "url": "https://..."
        }
      ]
    }
  }
}
```

**发现综合算法**：
1. 取 Top 3 结果
2. 去重（按 snippet 前100字符）
3. 拼接为可读摘要

### 3.4 调度器 (curious_agent.py)

**职责**：协调各组件，控制执行流程。

**交互模式**：
```
python3 curious_agent.py --run          # 手动运行一轮
python3 curious_agent.py --daemon      # 守护进程
python3 curious_agent.py --status      # 查看状态
python3 curious_agent.py --inject "X"  # 注入好奇心
```

**守护进程循环**：
```
while True:
    1. generate_initial_curiosities()  # 初始化队列
    2. rescore_all()                   # 重新评分
    3. select_next()                   # 选 Top 1
    4. explore()                       # 执行探索
    5. format_for_user()               # 格式化结果
    6. sleep(interval_minutes * 60)
```

## 四、接口设计

### 4.1 知识图谱 API

```python
# 读取
kg.get_state()                           # 获取完整状态
kg.get_top_curiosities(k=3)              # Top K 好奇心
kg.get_recent_knowledge(hours=24)        # 最近更新的知识
kg.get_knowledge_summary()                # 概览统计

# 写入
kg.add_knowledge(topic, depth, summary, sources)   # 添加知识
kg.add_curiosity(topic, reason, relevance, depth)  # 添加好奇心
kg.update_curiosity_status(topic, status)           # 更新状态
kg.log_exploration(topic, action, findings, notified)  # 记录探索
```

### 4.2 好奇心引擎 API

```python
engine = CuriosityEngine()
engine.generate_initial_curiosities()     # 生成初始队列
engine.rescore_all()                     # 重新评分所有
engine.select_next()                     # 选下一个
engine.add_contextual_curiosity(context) # 从上下文生成
```

### 4.3 探索器 API

```python
explorer = Explorer()
explorer.explore(curiosity_item)         # 执行探索，返回结果
explorer.format_for_user(result)         # 格式化为用户消息
```

## 五、数据流

```
[时间触发 / 手动触发]
        │
        ▼
[好奇心引擎.select_next()]
        │
        ▼
[探索器.explore(topic)]
    ├── Bocha API → 综合摘要
    └── 推理引擎 → 推导发现
        │
        ▼
[知识图谱.add_knowledge()]
        │
        ├── 好奇心分数 ≥ 7
        │       │
        │       ▼
        │   [飞书通知用户]
        │
        └── 好奇心状态 → done
        │
        ▼
[日志记录]
        │
        ▼
[等待下一轮]
```

## 七、设计权衡

| 决策 | 选择 | 理由 | 未来可改进 |
|------|------|------|-----------|
| 持久化格式 | JSON | 零依赖，可调试 | SQLite → Neo4j |
| 评分方式 | 启发式加权 | 可解释，快速迭代 | ML 模型训练 |
| 探索模式 | 串行单步 | MVP 简单可靠 | 并行多 Agent |
| 推理模式 | 基于已有知识 | 零 API 消耗 | LLM 深度推理 |
| 通知方式 | 飞书 | 已集成 | Slack / Email |

---

## 八、安全与边界

- **无外部写操作** — 不执行删除、发送等不可逆操作
- **API 限流** — Web Search 有 10s 超时
- **状态隔离** — 每个实例独立 state.json
- **日志可清理** — exploration_log 最多保留100条
