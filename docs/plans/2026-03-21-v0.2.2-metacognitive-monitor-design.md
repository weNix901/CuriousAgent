v0.2.1 的致命缺陷（2026-03-21 实测数据）：

```
最近 10 次探索日志：
  Topic: "Embodied Generative Cognitive"
  触发: 00:00 → 00:30 → 01:00 → ... → 04:00（连续10次）
```

## 二、理论基础

### 2.1 Flavell (1979) 元认知框架

```
元认知 = 监控（Monitoring）+ 调节（Control）

监控 → "我现在理解得怎么样？"  （信息从探索行为 → 元认知系统）
调节 → "我应该调整策略吗？"   （信息从元认知系统 → 探索行为）
```

**关键洞察**：监控和调节是**两个独立过程**，必须分离为独立模块。

### 2.2 Monitor-Generate-Verify 框架（arXiv:2510.16374, Oh & Gobet 2025）

```

│  │   Monitor    │───▶│   Generate   │───▶│   Verify    │  │
│  │   监控       │    │   生成       │    │   验证      │  │
```

## 四、核心模块详细设计

### 4.1 MetaCognitiveMonitor（纯监测）

```python
class MetaCognitiveMonitor:
    """
    元认知监控器 — 纯监测，不做决策
    
    职责：
    1. 探索计数（explore_counts）
    2. 边际收益计算（marginal_returns）
    3. 探索质量评分（last_quality）
    """
    
    def __init__(self, kg: KnowledgeGraph):
        self.kg = kg
        self.state = kg.load_state()
    
    def get_explore_count(self, topic: str) -> int:
        """获取话题已被探索的次数"""
        mc = self.state.get("meta_cognitive", {})
        counts = mc.get("explore_counts", {})
        return counts.get(topic, 0)
    
    def get_marginal_returns(self, topic: str) -> list:
        """获取话题的边际收益历史"""
        mc = self.state.get("meta_cognitive", {})
        returns = mc.get("marginal_returns", {})
        return returns.get(topic, [])
    
    def get_last_quality(self, topic: str) -> float:
        """获取话题最近一次探索质量分"""
        mc = self.state.get("meta_cognitive", {})
        qualities = mc.get("last_quality", {})
        return qualities.get(topic, 0.0)
    
    def assess_exploration_quality(self, topic: str, findings: dict) -> float:
        """
        评估探索质量（0-10分）
        
        公式：
        quality = new_discovery_rate × 0.4 
                + depth_improvement × 0.3 
                + user_relevance × 0.3
        
        其中：
        - new_discovery_rate: 新关键词数 / 总关键词数（避免重复探索）
        - depth_improvement: summary 质量提升（本次 vs 上次，0-1 归一化）
        - user_relevance: 与 USER_INTERESTS 的重叠度（0-1 归一化）
        
        Returns:
            quality: float, 0-10 分
        """
        pass  # OpenCode 实现
    
    def compute_marginal_return(self, topic: str, current_quality: float) -> float:
        """
        计算边际收益
        
        marginal_return = current_quality - previous_quality
        
        Returns:
            float: 边际收益（可为负）
```

### 4.2 MetaCognitiveController（纯控制）

```python
class MetaCognitiveController:
    """
    元认知控制器 — 基于 Monitor 输出做决策
    
    依赖 MetaCognitiveMonitor 的监测结果
    不直接访问知识图谱
    """
    
    def __init__(self, monitor: MetaCognitiveMonitor, config: dict = None):
        self.monitor = monitor
        self.config = config or {}
        
        # 决策阈值（可配置，通过 config 传入）
        self.thresholds = {
            "max_explore_count": 3,           # 同一话题最多探索次数
            "min_marginal_return": 0.3,       # 边际收益下限
            "high_quality_threshold": 7.0,     # 高质量通知阈值
        }
        self.thresholds.update(self.config.get("thresholds", {}))
    
    def should_explore(self, topic: str) -> tuple[bool, str]:
        """
        探索前检查：是否允许开始探索
        
        检查顺序：
        1. 探索计数是否已达上限
        2. 边际收益是否持续过低
        
        Returns:
            (allowed: bool, reason: str)
        """
        count = self.monitor.get_explore_count(topic)
        if count >= self.thresholds["max_explore_count"]:
            return False, f"已达最大探索次数({count}/{self.thresholds['max_explore_count']})"
        
        marginal = self.monitor.get_marginal_returns(topic)
        if len(marginal) >= 2:
            avg_recent = sum(marginal[-2:]) / 2
            if avg_recent < self.thresholds["min_marginal_return"]:
                return False, f"边际收益过低({avg_recent:.2f})"
        
        return True, "可以探索"
    
    def should_continue(self, topic: str) -> tuple[bool, str]:
        """
        探索后检查：同一话题是否继续探索
        
        检查：
        - 边际收益趋势是否仍为正
        
        Returns:
            (should_continue: bool, reason: str)
        """
        marginal = self.monitor.get_marginal_returns(topic)
        if len(marginal) >= 2:
            avg_recent = sum(marginal[-2:]) / 2
            if avg_recent < self.thresholds["min_marginal_return"]:
                return False, f"边际收益过低({avg_recent:.2f})"
        
        return True, "可以继续探索"
    
    def should_notify(self, topic: str) -> tuple[bool, str]:
        """
        探索后检查：是否通知用户
        
        规则：
        - 质量分数 >= high_quality_threshold → 高质量发现
        
        Returns:
            (should_notify: bool, reason: str)
        """
        quality = self.monitor.get_last_quality(topic)
        
        if quality >= self.thresholds["high_quality_threshold"]:
            return True, f"高质量发现({quality:.1f})"
        else:
            return False, f"质量不足({quality:.1f})"
```

## 五、与 CuriousAgent 的集成

### 5.1 curious_agent.py 修改

```python
class CuriousAgent:
    def run_one_cycle(self, topic: str):
        # === 初始化 Monitor ===
        monitor = MetaCognitiveMonitor(self.kg)
        controller = MetaCognitiveController(monitor)
        
        # === Monitor 阶段：探索前检查 ===
        allowed, reason = controller.should_explore(topic)
        if not allowed:
            logger.info(f"探索被阻止: {topic} — {reason}")
            # 标记为完成（不再重复探索）
            self.kg.mark_topic_done(topic)
            return
        
        # === Generate 阶段 ===
        explorer = Explorer(exploration_depth=self.depth)
        findings = explorer.explore(topic)
        
        # === Monitor 评估 ===
        quality = monitor.assess_exploration_quality(topic, findings)
        marginal = monitor.compute_marginal_return(topic, quality)
        monitor.record_exploration(topic, quality, marginal, notified=False)
        
        # === Verify 阶段 ===
        should_notify, notify_reason = controller.should_notify(topic)
        if should_notify:
            self.notify_user(topic, findings, quality)
            # 更新 notified 标记
            self.kg.update_last_exploration_notified(topic, True)
        
        # === 继续决策 ===
        continue_allowed, continue_reason = controller.should_continue(topic)
        if continue_allowed:
            logger.info(f"继续探索: {topic} — {continue_reason}")
            # 同一话题加入队列下一轮
            self.kg.add_curiosity(topic, score=quality, reason=f"边际收益:{marginal:.2f}")
        else:
            logger.info(f"结束探索: {topic} — {continue_reason}")
            self.kg.mark_topic_done(topic)
```

### 5.2 关键行为变化

| 场景 | v0.2.1 行为 | v0.2.2 行为 |
|------|------------|------------|
| 同一话题探索第 4 次 | 继续探索 | **被阻止**，标记 done |
| 边际收益 < 0.3 | 继续探索 | **停止**，标记 done |
| 探索质量 >= 7.0 | 通知用户 | 通知用户（不变） |
| 探索质量 < 7.0 | 可能通知 | **不通知**，静默记录 |

| `record_exploration(topic, quality, marginal, notified)` | 同上 | None | 写入 state.json |
| `should_explore(topic)` | topic: str | (bool, str) | 无 |
| `should_continue(topic)` | topic: str | (bool, str) | 无 |
| `should_notify(topic)` | topic: str | (bool, str) | 无 |

### 9.3 决策真值表

| 探索次数 | 边际收益趋势 | 质量分 | should_explore | should_continue | should_notify |
|---------|------------|--------|---------------|----------------|--------------|
| 0 | — | — | ✅ | — | — |
| 1 | 高(0.9) | 8.0 | ✅ | ✅ | ✅ 通知 |
| 2 | 低(0.1) | 6.0 | ✅ | ❌ 停止 | ❌ 不通知 |
| 3 | -0.2 | 4.0 | ❌ 阻止 | ❌ 停止 | ❌ 不通知 |
| 4+ | 任意 | 任意 | ❌ 阻止 | ❌ 停止 | ❌ 不通知 |

### 9.4 错误处理

| 场景 | 处理方式 |
|------|---------|
| `get_explore_count` 话题不存在 | 返回 0 |
| `compute_marginal_return` 无历史数据 | 返回 1.0（首次探索默认高收益） |
| `assess_exploration_quality` 失败 | 返回 5.0（默认中等质量） |
| `record_exploration` 写入失败 | 抛出异常，上层捕获并 log |
| state.json 损坏/无 meta_cognitive 字段 | 初始化空结构 `{"explore_counts": {}, ...}` |
